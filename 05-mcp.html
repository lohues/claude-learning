Module 05 · power with a standing tax

Connecting the outside world —
and knowing when not to.

MCP lets Claude reach GitHub, databases, browsers, design tools. It’s genuinely powerful. It also taxes the context window you just spent Module 03 learning to protect — every turn, whether you use the server or not. So the core skill here isn’t connecting. It’s deciding.

“Should I connect this?” — answer threeest. always-on cost: —
Q1How often will you use this integration?
Q2How many tools does the server expose?
Q3Where’s the server from?

MODULE 05

MCP & the Outside World

● core ~27 min read /chrome is beta 6 lessons · 1 check

What MCP is, how to connect a server (transports & scope), the standing context tax that makes restraint the real skill, when to reach for a CLI instead, the hygiene that keeps it safe, and the browser as a tool via /chrome.

5.1 · What MCP is

The Model Context Protocol: a standard way to plug external systems into Claude as callable tools.

Skills (Module 04) gave Claude procedures. MCP gives it reach — beyond your filesystem to GitHub, Postgres, Slack, Figma, your browser. An MCP server exposes a set of tools (e.g. “create issue,” “run query”); Claude calls them like any other tool. One protocol, many integrations, no bespoke glue per service.

5.2 · Connecting a server: transports & scope

One command, two decisions: how it connects (transport) and who sees it (scope).

claude mcp add
# flags BEFORE the name; '--' separates the server's own command
$ claude mcp add --transport stdio my-db -- \
      npx -y @modelcontextprotocol/server-postgres postgres://localhost/mydb

$ claude mcp add --transport http linear https://mcp.linear.app/mcp

> /mcp        # live status + run OAuth for servers that need it

Transports:

transport
use
stdio
Local process on your machine. Good for local DBs, filesystem-adjacent tools.
http
Remote server. The current standard for hosted integrations.
sse
Deprecated. If you see it, migrate to http.

Scope decides where the config lives and who inherits it: local (default, ~/.claude.json, just you) · project (.mcp.json, committed and shared with the team) · user (all your projects).

5.3 · The tax: every server costs, every turn

This is the lesson that changes how you use MCP. A connected server injects its entire tool schema into context on every turn — used or not.

Remember the context gauge from Module 03? MCP tool definitions were the single biggest surprise consumer. Here’s why, in real numbers:

  • Connecting four servers can burn ~7,000 tokens before you type a single prompt.
  • A heavy configuration has been measured at ~66,000 tokens — roughly a third of a 200K window — gone to definitions you may never call.
  • GitHub’s official server (94 tools) alone runs about 17,600 tokens per request just for its tool definitions.
The reframe

An idle MCP server isn’t free — it’s a standing cost, paid on every turn for the whole session. The question is never “could this server be useful?” (almost anything could). It’s “is it useful often enough to justify taxing every turn?” That’s what the engine up top computes.

5.4 · When NOT to use MCP

Three situations where connecting is the wrong move.

  • One-off tasks. Need to check one AWS resource? Don’t mount a 60-tool server for the whole session. Claude can call the aws or gh CLI through Bash — you pay once, nothing lingers.
  • Occasional use of a big surface. A CLI you invoke when needed beats a permanent definitions tax. CLIs cost tokens only when run.
  • Noisy outputs. An MCP tool that dumps 10,000 lines of JSON poisons the rest of the session — that output sits in context degrading every later turn. Prefer tools (or CLIs) with tight, summarizable output.
Rule of thumb

CLI for occasional, MCP for constant. If you’d use it many times a session, the standing tax amortizes and MCP wins on ergonomics. If you’d use it once, the CLI wins on context. The middle is where Tool Search comes in (next).

5.5 · Mitigations & hygiene

If you do connect servers, these keep the tax and the risk down.

  • Tool Search. Enable it (ENABLE_TOOL_SEARCH) and tool definitions load on demand once they cross a size threshold, instead of squatting in context every turn. This is what makes a 90-tool server tolerable.
  • Disable what you’re not using. Check /mcp and turn off servers you haven’t touched in two weeks. Idle ≠ free.
  • Read-only credentials. Give a server the least access that works. A query server doesn’t need write keys.
Security · audit before you connect

A third-party MCP server runs with your access and can see what Claude sends it. Treat connecting one like npm install-ing a dependency you didn’t read: audit the source, prefer official/first-party servers, and assume a compromised server is a path to your data. This pairs with Module 02 — a deny rule on .env still holds regardless of what a server requests.

5.6 · /chrome — the browser as a tool

A built-in path to the outside world that earns its own command: your actual browser.

The /chrome integration connects Claude Code to the Claude-in-Chrome extension and your logged-in browser session. Claude can navigate, screenshot, read the console, inspect the DOM, and record GIFs. That unlocks:

  • Live debugging — read a console error, fix the code, reload, confirm.
  • Design verification — compare a rendered page against a mockup.
  • Form / flow testing and visual regression across changes.
Gotcha · it’s beta, and picky

/chrome is beta: Chrome/Edge only, and not supported on WSL, Brave, or Arc. It shares your real browser state, so be deliberate about what’s logged in. For unattended CI-style automation, a Playwright or Chrome-DevTools MCP server is the better fit. The reason it matters here: /chrome closes the loop that becomes Module 06’s self-correcting verification — edit, screenshot, detect, fix, with no human in the middle.

  • Do a one-off task with a CLI via Bash instead of a servere.g. gh pr list rather than the GitHub MCP
  • Turn on Tool Search and re-measure a heavy server’s costwatch idle definitions drop
  • Connect /chrome and have Claude screenshot a local pageskip if you’re on WSL
